# Business OS 产品设计概要

> 生意Skill自生成平台：让企业经营方法论变成可执行的AI原生工作系统

---

## 一、产品背景

产业带中小企业（如义乌小商品、深圳3C配件等）面临的核心问题：

- 老板脑子里有经营方法论（选品逻辑、定价策略、客户跟进节奏），但无法系统化沉淀
- 每个企业的做法不同，通用SaaS无法覆盖
- 日常重复性工作占据大量时间（询盘回复、数据复盘、客服考核等）
- 缺乏数字化能力，经营决策依赖经验和直觉

## 二、产品愿景

**用 Skill 承接企业工作流，通过对话沉淀 → 自动执行 → 数字孪生的三阶段演进，让中小企业拥有AI原生的经营操作系统。**

### 演进路径

```
Phase 1: 对话沉淀
  用户通过对话描述自己的工作方式 → 系统生成可执行的 Skill

Phase 2: 自动运转
  Skill 按计划/事件自动执行 → 日常任务自转 → 用户只看结果和异常

Phase 3: 数字孪生
  系统持有企业运营全貌 → 主动发现问题 → 辅助决策 → 企业数字孪生
```

## 三、核心概念：Skill

Skill 是平台的核心载体，它封装了一套完整的企业经营方法论：

### Skill 四件套

| 组件 | 说明 | 示例 |
|------|------|------|
| **skill.yaml** | 方法论定义（步骤、提示词、工具、检查点） | 爆款选品分析的3步流程 |
| **reference/** | 参考文档（每步的细化规则） | 选品评分标准、竞品分析维度 |
| **tools** | 执行工具（取数、计算、生成） | 查询订单数据、计算利润率 |
| **assets/** | 企业资产（私有数据、偏好） | 店铺品类范围、历史爆款数据 |

### Skill YAML 结构

```yaml
id: new_product_plan
name: 爆款选品分析
description: 基于市场趋势和竞品分析，找出潜力爆款方向
version: 1

intake:
  - id: category
    question: 你想分析哪个品类？
  - id: budget
    question: 预算范围？
  - id: preference
    question: 偏好什么风格？

steps:
  - id: market_scan
    name: 市场趋势扫描
    prompt: 分析 {category} 品类的市场趋势...
    tools: [query_products, query_traffic]
  - id: competitor
    name: 竞品深度分析
    prompt: 对比TOP竞品...
    tools: [query_competitor]
    checkpoint: true  # 需用户确认后继续
  - id: recommendation
    name: 选品推荐
    prompt: 综合以上分析，推荐3-5个潜力方向...

system_prompt: |
  你是一位资深电商选品专家...
  {enterprise_assets}

capture_prompt: |
  执行完成。需要保存你的品类偏好和分析模板吗？
```

## 四、交互设计

### 核心交互：引导式多轮对话

不是"用户问一句、AI答一段"的报告模式，而是**引导式的、有状态的多轮交互**：

```
用户: 帮我做个选品分析
   ↓
系统: [识别意图 → 匹配 Skill]
系统: 好的！我来帮你做选品分析。先回答几个问题：
系统: 1. 你想分析哪个品类？
   ↓
用户: 家居收纳
   ↓
系统: 2. 预算范围？
   ↓
用户: 10-30元
   ↓
系统: 收到。我的执行计划：
      Step 1: 扫描家居收纳品类趋势
      Step 2: 分析TOP5竞品（含数据拉取）
      Step 3: 给出3-5个选品方向建议
      确认执行？
   ↓
用户: 确认
   ↓
系统: [执行 Step 1...] 📊 市场趋势报告...
系统: [执行 Step 2...] 🏪 竞品分析...
系统: ⚠️ Checkpoint: 竞品分析完成，以下是发现：
      - 竞品A主打低价，月销5000+
      - 竞品B主打设计感，评价4.8
      需要基于哪个方向深入分析？
   ↓
用户: 设计感方向
   ↓
系统: [执行 Step 3...] 🎯 推荐方向：
      1. 莫兰迪色系旋转收纳盒
      2. ...
      
系统: 📝 要保存你的偏好吗？
      - 品类偏好: 家居收纳
      - 风格偏好: 设计感/莫兰迪色系
      - 价格带: 10-30元
   ↓
用户: 保存
```

## 五、功能清单

### P0 — 核心循环（让产品能"自动跑起来"）

| 功能 | 说明 | 状态 |
|------|------|------|
| 交互式 Skill 执行 | 引导问答 → 确认计划 → 分步执行 → 偏好沉淀 | ✅ |
| 对话式 Skill 创建 | 用自然语言描述，AI 自动生成 YAML | ✅ |
| 企业上下文注入 | 执行时自动加载企业资产与偏好 | ✅ |
| 执行历史 | 记录输入、输出、耗时，可回看 | ✅ |
| 企业记忆持久化 | 偏好/决策被记住，下次自动应用 | ✅ |
| 定时自动执行 | Cron/间隔调度，自动运行 Skill | ✅ |

### P1 — 产品深度（让每个 Skill 真正有用）

| 功能 | 说明 | 状态 |
|------|------|------|
| 更多预装 Skill | 退款分析、客户分群、定价策略 | ✅ (共5个) |
| Tool 系统 | 可扩展的工具注册表 + 10个工具 | ✅ |
| 文件上传 | CSV/Excel/图片等企业资产上传 | ✅ |
| 执行结果导出 | 对话/结果一键导出 Markdown | ✅ |

### P2 — 产品广度（让产品更完整）

| 功能 | 说明 | 状态 |
|------|------|------|
| 实时通知推送 | SSE 推送，Skill完成即时提醒 | ✅ |
| 数据看板 | 资产、执行、对话核心指标 | ✅ |
| 电商平台对接 | Mock 数据层，预留真实 API | ✅ |
| 团队协作 | 成员管理、角色、审计日志 | ✅ |

### P3 — 长远目标（向数字孪生演进）

| 功能 | 说明 | 状态 |
|------|------|------|
| Skill 市场 | 发布/搜索/安装共享 Skill | ✅ |
| 移动端适配 | 响应式布局 + 移动端侧滑 | ✅ |
| 数字孪生仪表盘 | 五维状态模型可视化 + 健康度 | ✅ |

## 六、预装 Skill 一览
